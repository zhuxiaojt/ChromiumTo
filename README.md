ChromiumTo是一款专为Chromium内核浏览器打造的共享工具，能够让您设备上的多个Chromium内核应用（如各类基于Chromium开发的浏览器）共享同一个内核文件，无需重复占用磁盘空间，同时实现内核版本的统一管理，一次更新即可覆盖所有关联应用。

## 核心优势

 - 节省磁盘空间：多个浏览器共享同一个内核文件，大幅减少冗余存储占用
 - 统一版本管理：只需更新共享内核，即可同步更新所有关联浏览器版本
 - 简单易用：图形化操作界面，无需专业技术，新手也能快速上手
 - 安全可靠：重定向前自动备份原始文件，可随时恢复至初始状态，无风险隐患

## 核心功能

#### 智能系统扫描

支持快速扫描（常见安装目录）和全盘扫描（全磁盘分区），自动识别所有Chromium内核应用，无需手动查找。

#### 灵活内核配置

支持从现有应用初始化、自动下载最新内核、手动指定内核路径三种方式，灵活设置共享内核。

#### 批量重定向

支持单个/全部应用批量重定向至共享内核，操作高效，节省时间成本。

#### 安全恢复机制

已重定向应用可随时单个/批量恢复至原始内核状态，自动备份文件不丢失，使用更安心。

#### 轻松删除备份

你可以直接在界面中删除备份，无需额外操作。

#### 实时进度显示

扫描和下载过程中实时展示进度状态，让您清晰掌握当前任务推进情况，无需盲目等待。

#### 空间节省计算

自动计算并展示通过共享内核节省的磁盘空间，让您直观看到产品带来的实际价值。

## 常见问题

#### 重定向失败怎么办？

重定向失败可能有以下原因：

 - 没有足够的权限创建符号链接，请尝试以管理员身份运行应用
 - 目标文件正在被使用，请关闭相关浏览器后重试
 - 共享内核路径不存在或不完整，请检查共享内核设置

#### 如何卸载ChromiumTo？

卸载ChromiumTo前，请先将所有应用恢复到原始状态，然后删除应用文件即可。

#### 扫描结果中包含我设置的共享内核，这正常吗？

是的，这是正常现象。应用会将共享内核识别为"作为共享内核使用"，以便您了解当前的共享内核状态。

#### 为什么有些浏览器没有被扫描到？

可能是因为这些浏览器安装在非标准位置，或者使用了特殊的安装方式。您可以尝试使用"全盘扫描"来查找更多浏览器。

#### ChromiumTo支持哪些系统？

目前ChromiumTo主要针对Windows操作系统进行了深度适配与优化，能够为Windows系列系统（包括Windows 10、Windows 11等主流版本）提供稳定、高效的Chromium内核共享服务，暂未对其他操作系统提供支持哦。

#### 为什么重定向后没有节省磁盘空间？

为了安全起见，ChromiumTo在重定向应用时，会自动备份原始内核文件，确保您可以随时恢复。如果你确认重定向后应用正常运行，可以手动删除备份文件，释放磁盘空间。

#### 为什么自动下载共享内核会失败？

共享内核使用官方Chromium下载链接，依赖国外服务器，可能会因为连接超时导致下载失败。您可以尝试手动下载共享内核、使用代理服务器或检查网络连接。
